# WhatWhenWhere

Python prototype for remote image capture, location/orientation upload, server-side organization, and image-analysis workflows.

The original system concept used a Raspberry Pi client to send timestamped images plus latitude, longitude, pitch, roll, and heading data to a server. The server side organized submitted image sets and supported image-analysis and review workflows.

## What It Shows

- Python hardware-adjacent capture workflow
- Client/server requirements and design thinking
- Image, GPS, and orientation metadata modeling
- ML-oriented image-analysis workflow planning
- Browser-facing review surface concept
